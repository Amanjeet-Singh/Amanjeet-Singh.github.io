Accessing Websites

An explanation of how websites are accessed

Now that we know what a website, or more specifically, what a webpage is, things are a tiny bit clearer, but we still have a long way to go. These webpages still have to get to the browser somehow, to be displayed on our screens, but how do they get to the browser? Are all the websites of the world stored in our own computers in html files that we can access? or are they being sent to our browser through the internet from somewhere? Where are they being sent from? How are they being sent to us? These questions are vital to understanding how websites are accessed and displayed in our browsers and below I will answer them.

The Server

A completed webpage in the form of an html file and all the other files that it needs, is stored in a server. A server exists for the sole purpose of storing information and, in the case of web servers, making it available on the internet. These servers are usually really fast in computing speed and have access to really fast internet connections so that the webpages they contain can be sent to our browser as fast as possible.

DNS and IP Adresses

A plethora of information is sent and recieved on the internet each day, and among this stream, we have to be sure that we are only accessing the websites we want to go to, and for this, a robust system must be put in place so that each website has a unique identity that can be used to separate it from others. Each server and website has a unique IP(Internet Protocol) address which is used to identify the server and the website. But, we don't type in IP addresses in the browser, we type in URLs, so how can we still access these websites? Well, the IP address of most public websites are connected to domain names, a.k.a the URLs of these webpages. These connections are stored as information in a DNS (Domain Name Server). The DNS is a server which tells the browser what IP address does a domain name correspond to.

It All Comes Together!

These are the all the basic elements involved in getting a webpage from a browser to your screen. The process is as follows:

  1. User enters a URL into the browser, or clicks on a link which inputs the URL of a website/page automatically
  2. The browser sends this url to a DNS
  3. The DNS checks its records to find which IP address the particular domain name corresponds to and sends this IP address back to the browser
  4. The browser then contacts the server through the IP address, and requests the source code for the website/page it wants to display.
  5. The server sends back source code for the page and all other linked files that are needed for the page to load
  6. Finally, the browser loads the source code and all other files and interprets it, and displays the design and content according to what the source code said, and voila! a webpage is displayed.
