CSS

Examples and uses of CSS

Displaying HTML code by itself is not a very pretty website. We solve this by writing Cascading Style Sheets, or CSS, so we can style our page. A website of today needs to have CSS styling in order to look good. The difference can be astonishing to look at.

CSS uses

CSS allows us to change almost anything about an element, from its size, shape, borders, color, to what it does when its viewed on a smaller scree, or when it hovered over, etc. It is the magic by which modern Navbars look the way they do, while structurally being not more than unordered list in our HTML source code. To style any HTML element with CSS, we can either do an inline styling inside the HTML file, or create a new CSS file and link it to our HTML page. The latter is usually the one recommended as it allows us do to much more than inline styles and helps keep code de-cluttered.

CSS Syntax

CSS has some syntax rules that we must follow to style elements. First of all, most elements that you are styling need to be clearly declared, with the name of that element/class/id, and then withing a set of curly brackets, the styling statements can be written, but they have to be ended with a semicolon. To declare styling for a HTML element globally (such as styling for all the <p> elements in a page, we only need to write the letter p and fit in the styling in the curly brackets that come after. For a particular, a period '.' must be added before the class name, and for particular ids, a '#' must be added before the id name.
