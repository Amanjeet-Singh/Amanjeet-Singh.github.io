Bootstrap

Overview of Bootstrap and some common classes

As we discussed earlier, Bootstrap is a framework for HTML/CSS that makes a developer's job much easier and less time-consuing. Because of the functionality that Bootstrap and other such frameworks provide, developers don't have to structure and style from scratch every element in their websites, and this can prove to be extremely useful in dynamic sites with animations, and loads of data.

What Bootstrap does

Bootstrap comes with a library of pre-defined classes that we can just attribute to any element on our page to make it look and act how the pre-defined class specifies. For example, instead of manually writing out the css for our navbar, buttons, and forms, etc. we can just add the pre-defined bootstrap classes to them, making them responsive, and look the way we want to, but with much less work. To use bootstrap, a developer needs to have the bootstrap files that define these classes in his server and link those files to the page, or link a CDN that provides these files online.

Bootstrap Grid and Classes

Other than being able to bring design to elements by simply adding classes, bootstrap also has a very nifty feature that makes structuring much easier. We can use bootstrap classes to divide up our page into invisible rows and columns, and then put content into them. This makes the job of structuring content position on the page much lighter, and you don't have to mess with margins, padding, or alignment in the css file to achieve it.

Bootstrap has a large number of classes that can be utilized by developers. These same classes can be made responsive, can be made functional by JavaScript, and can be colored by using a '-' after the class and with another class specification, to make things easier. All the styling we do from Bootstrap can be overrided by CSS if we wish to customize some elements more than what Bootstrap offers.
